# user 부하 테스트

## 테스트 대상 API

| 메서드  | 경로                   | 설명                               |
|------|----------------------|----------------------------------|
| POST | `/api/v1/auth/login` | 이메일/패스워드 로그인 → JWT 발급 + Redis 저장 |

## 시나리오

### smoke
- 2 VU / 30초
- 로그인 정상 응답(200) 확인

### load
- 0 → 50 VU ramping (30s 증가 → 1분 유지 → 30s 감소)
- 다수 유저 동시 로그인 반복
- JWT 발급 처리량 및 Redis 쓰기 부하 측정

### spike
- 0 → 50 VU 순간 급증 (10초 내 최대치)
- 갑작스러운 동시 로그인 집중 상황 시뮬레이션

## 핵심 관찰 포인트

- 로그인 응답 시간 분포 (p95, p99)
- 동시 접속자 증가 시 Redis 쓰기 지연 여부
- spike 구간에서 오류율 증가 여부

## 메트릭

| 메트릭                      | 임계값                            |
|--------------------------|--------------------------------|
| `user_login_duration_ms` | p(95) < 1000ms, p(99) < 2000ms |
| `user_login_error_rate`  | rate < 5%                      |
